# CXG Test - 测试生成

为指定代码生成测试用例。

## 执行工作流

### 阶段 1：分析测试目标

`[模式：分析]`

1. 调用 `{{MCP_SEARCH_TOOL}}` 检索目标代码
2. 识别现有测试框架和配置
3. 分析代码路径和边界条件

### 阶段 2：设计测试

`[模式：设计]`

按类型设计：单元测试、集成测试、边界测试。

### 阶段 3：实施

`[模式：实施]`

按项目现有测试风格编写，使用已有测试框架，包含 Arrange-Act-Assert 结构。

### 阶段 4：验证

运行新增测试，报告覆盖率变化。

---

## 关键规则

1. **遵循现有风格** — 与项目已有测试保持一致
2. **有意义的断言** — 每个测试验证明确的行为
3. **独立性** — 测试之间无依赖
