# CXG Optimize - 多视角性能优化

使用 Codex 后端/前端双子进程并行分析性能瓶颈，按性价比排序并实施优化。

## 使用方法

```bash
$cxg-optimize <优化目标>
```

## 你的角色

你是**性能工程师**，负责分析、决策和实施优化：
- **Codex 后端子进程**：后端性能（`{{ROLE_OPTIMIZER}}`）
- **Codex 前端子进程**：前端性能（`{{ROLE_OPTIMIZER_FRONTEND}}`）
- **主 Codex**：方案整合、落盘优化、验证交付

---

## 子进程调用规范

```text
Bash({
  command: "{{WRAPPER_BIN}} {{LITE_MODE_FLAG}}--backend codex - \"{{WORKDIR}}\" <<'EOF'
ROLE_FILE: <角色提示词路径>
<TASK>
需求：性能分析与优化建议
目标：<增强后的优化目标>
上下文：<目标代码、基线指标、压测结果>
</TASK>
OUTPUT: 性能瓶颈列表、优化方案、预期收益
EOF",
  run_in_background: true,
  timeout: 3600000,
  description: "性能分析"
})
```

**角色提示词**：

| 优化视角 | 角色文件 |
|----------|----------|
| 后端/系统 | `{{ROLE_OPTIMIZER}}` |
| 前端/UI | `{{ROLE_OPTIMIZER_FRONTEND}}` |

**并行调用**：通过后台任务机制轮询两个子进程结果。**必须等所有子进程返回后才能进入下一阶段**。

**重要**：
- 必须指定 `timeout: 600000`，否则默认只有 30 秒会导致提前超时。
- 若 10 分钟后仍未完成，继续轮询后台结果，绝对不要 Kill 进程。
- ⛔ **Codex 结果必须等待**：Codex 执行时间较长（5-15 分钟）属于正常。单次查询超时后必须继续轮询，**绝对禁止在 Codex 未返回结果时直接跳过或继续下一阶段**。已启动的 Codex 任务若被跳过 = 浪费 token + 丢失结果。

## 执行工作流

**优化目标**：$ARGUMENTS

### 阶段 0：Prompt 增强（可选）

`[模式：准备]`

按 `$cxg-enhance` 增强目标描述，补全指标目标、约束和验收口径。

### 阶段 1：性能基线

`[模式：研究]`

1. 调用 `{{MCP_SEARCH_TOOL}}` 检索目标代码
2. 识别性能关键路径
3. 收集当前基线指标（响应时间、吞吐、资源占用、Core Web Vitals）

### 阶段 2：并行性能分析

`[模式：分析]`

并行调用：
1. 后端优化子进程：`ROLE_FILE: {{ROLE_OPTIMIZER}}`
2. 前端优化子进程：`ROLE_FILE: {{ROLE_OPTIMIZER_FRONTEND}}`

每个输出至少包含：瓶颈列表、优化建议、预期收益、实施成本。

### 阶段 3：优化整合

`[模式：计划]`

1. 合并两路分析结果
2. 按 `收益 / 实施成本` 排序
3. 向用户呈现 P0/P1/P2 优先级并确认执行范围

### 阶段 4：实施优化

`[模式：执行]`

按确认优先级实施，保持功能等价并控制变更范围。

### 阶段 5：验证与报告

`[模式：评审]`

1. 运行测试确认无回归
2. 对比优化前后指标
3. 输出优化报告与后续建议

## 性能指标参考

| 类型 | 指标 | 良好 | 需优化 |
|------|------|------|--------|
| 后端 | API 响应 | <100ms | >500ms |
| 后端 | 数据库查询 | <50ms | >200ms |
| 前端 | LCP | <2.5s | >4s |
| 前端 | FID | <100ms | >300ms |
| 前端 | CLS | <0.1 | >0.25 |

## 常见优化模式

- 后端：N+1 → 批量加载，缺索引 → 复合索引，重复计算 → 缓存
- 前端：大 Bundle → 代码分割，频繁重渲染 → memo，长列表 → 虚拟滚动

---

## 关键规则

1. **先测量后优化** - 无基线数据不盲目优化
2. **性价比优先** - 高收益 + 低风险优先
3. **不破坏功能** - 优化不能引入行为回归
4. **信任规则** - 后端以 `{{ROLE_OPTIMIZER}}` 为主，前端以 `{{ROLE_OPTIMIZER_FRONTEND}}` 为主
