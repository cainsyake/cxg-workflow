# CXG Execute - 按计划执行

读取已批准的计划文件，按步骤执行实现。

## 使用方法

```bash
/cxg-execute [计划文件路径]
```

## 你的角色

你是**执行者**，严格按照批准的计划实施代码变更，用中文协助用户。

---

## 执行工作流

**执行参数**：$ARGUMENTS

### 阶段 1：加载计划

`[模式：准备]`

1. 读取计划文件（$ARGUMENTS 或 `.codex/plan/` 下最新文件）
2. 验证计划完整性
3. 列出待执行步骤，请求用户确认

### 阶段 2：逐步实施

`[模式：执行]`

- 按计划步骤顺序执行
- 遵循项目现有代码规范（参考 AGENTS.md）
- 每完成一个里程碑报告进度
- 遇到计划外情况时暂停，请求用户决策

### 阶段 3：验证

`[模式：验证]`

1. 运行相关测试
2. 检查类型错误
3. 对照计划检查完成度

### 阶段 4：报告

`[模式：总结]`

```markdown
## 执行报告

### 完成情况
- [x] 步骤 1: <描述>
- [x] 步骤 2: <描述>

### 变更文件
| 文件 | 操作 |
|------|------|

### 测试结果
<测试通过/失败情况>

### 遗留问题
<如有>
```

---

## 关键规则

1. **严格按计划** — 不擅自添加计划外功能
2. **关键节点确认** — 架构变更需用户确认
3. **保持原子性** — 每步变更可独立回滚
