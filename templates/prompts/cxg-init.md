# CXG Init - 初始化项目上下文

以「根级简明 + 模块级详尽」策略生成项目 AGENTS.md 上下文文档。

## 使用方法

```bash
/cxg-init [项目摘要]
```

---

## 执行工作流

**项目摘要**：$ARGUMENTS

### 阶段 1：项目扫描

`[模式：扫描]`

1. 识别项目类型（语言、框架、构建工具）
2. 统计文件和目录结构
3. 识别入口文件和核心模块

### 阶段 2：模块分析

`[模式：分析]`

对每个核心模块：
- 入口文件
- 对外接口
- 关键依赖
- 测试覆盖

### 阶段 3：生成 AGENTS.md

`[模式：生成]`

**根目录 AGENTS.md**：

```markdown
# <项目名> — <一句话描述>

## 架构总览
<Mermaid 结构图>

## 模块索引
| 模块 | 职责 | 文档 |
|------|------|------|

## 运行与开发
### 常用命令
### 测试策略

## 编码规范
<从现有代码推断>
```

**模块级 AGENTS.md**（每个核心模块一个）：

```markdown
# <模块名>

## 职责
## 入口
## 对外接口
## 依赖关系
## 相关文件
```

### 阶段 4：输出摘要

`[模式：报告]`

```markdown
## 初始化结果

### 文档
- 根级 AGENTS.md: [创建/更新]
- 模块级 AGENTS.md: X 个

### 覆盖率
- 扫描文件: X / Y
- 覆盖模块: X%

### 推荐下一步
- [ ] <补充建议>
```

---

## 关键规则

1. **只写文档** — 不修改源代码
2. **忽略生成物** — 跳过 node_modules、dist、二进制文件
3. **增量更新** — 已有 AGENTS.md 时做更新而非覆盖
