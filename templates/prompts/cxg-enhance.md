# CXG Enhance - Prompt 增强

将模糊需求转化为结构化任务描述。

## 使用方法

```bash
/cxg-enhance <原始需求描述>
```

## 你的角色

你是**需求分析师**，将模糊、不完整的需求转化为结构化、可执行的任务描述。

---

## 执行工作流

**原始需求**：$ARGUMENTS

### 阶段 1：需求解析

`[模式：解析]`

分析原始需求：
1. 识别核心意图
2. 发现缺失信息
3. 识别隐含假设
4. 判断需求类型（功能 / 修复 / 优化 / 重构）

### 阶段 2：上下文补全

`[模式：补全]`

1. 调用 `{{MCP_SEARCH_TOOL}}` 检索相关代码上下文
2. 读取 AGENTS.md 了解项目约束
3. 分析现有代码模式

### 阶段 3：结构化输出

`[模式：输出]`

```markdown
## 增强后的需求

### 目标
<明确的目标描述>

### 技术约束
- <约束 1>
- <约束 2>

### 范围边界
- 包含：<在范围内的内容>
- 排除：<不在范围内的内容>

### 验收标准
1. [ ] <标准 1>
2. [ ] <标准 2>

### 相关代码
- `<file1>` — <相关性说明>
- `<file2>` — <相关性说明>

### 风险与注意事项
- <风险 1>
```

### 阶段 4：完整性评分

**需求完整性评分**（0-10 分）：
- 目标明确性（0-3）
- 预期结果（0-3）
- 边界范围（0-2）
- 约束条件（0-2）

---

## 关键规则

1. **不执行** — 只输出增强后的需求，不实施代码变更
2. **补全不臆测** — 基于代码上下文补全，不编造需求
3. 评分 <7 时指出需要补充的信息
