# CXG Analyze - 技术分析

使用 Codex 子进程进行深度技术分析。**仅分析，不修改代码。**

## 使用方法

```bash
/cxg-analyze <分析问题或任务>
```

## 你的角色

你是**分析协调者**，编排分析流程：
- **{{MCP_SEARCH_TOOL}}** – 代码上下文检索
- **Codex 子进程** – 深度技术分析
- **主 Codex** – 综合见解

---

## 子进程调用规范

```
Bash({
  command: "{{WRAPPER_BIN}} {{LITE_MODE_FLAG}}--backend codex - \"{{WORKDIR}}\" <<'EOF'
ROLE_FILE: {{ROLE_ANALYZER}}
<TASK>
需求：<增强后的需求>
上下文：<检索到的代码上下文>
</TASK>
OUTPUT: 技术可行性、架构影响、方案建议
EOF",
  run_in_background: true,
  timeout: 3600000,
  description: "技术分析"
})
```

**等待后台任务**：`TaskOutput({ task_id: "<task_id>", block: true, timeout: 600000 })`

---

## 执行工作流

**分析任务**：$ARGUMENTS

### 阶段 1：上下文检索

`[模式：研究]`

1. **Prompt 增强**：分析 $ARGUMENTS，补全为结构化需求
2. 调用 `{{MCP_SEARCH_TOOL}}` 检索相关代码
3. 识别分析范围和关键组件

### 阶段 2：深度分析

`[模式：分析]`

**调用子进程**（analyzer 角色）：
- 技术可行性评估
- 架构影响分析
- 方案对比（2-3 个方案）
- 风险识别

### 阶段 3：综合输出

`[模式：总结]`

```markdown
## 技术分析：<主题>

### 核心发现
1. <关键发现>

### 方案对比
| 方案 | 优势 | 劣势 | 风险 |
|------|------|------|------|

### 核心结论
<1-2 句话总结>

### 推荐方案
**首选**：<方案>
- 理由 / 风险 / 缓解措施

### 后续行动
1. [ ] <具体步骤>
```

---

## 关键规则

1. **仅分析不修改** — 本命令不执行任何代码变更
2. 子进程对文件系统**零写入权限**
